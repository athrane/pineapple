com.alpha.pineapple.session description

Content

  • Session management
    • Contains implementation of SessionManager which manages session with external resources.

Class diagram

Interfaces and classes

PluginActivator

Defines the session management interface for the application. Provides access to external resources through sessions.

PluginActivatorlmpl

Implementation of the SessionManager interface.

PluginActivator instances are created by ManagerImpl class during initialization of the core component. The initialization is implemented in the command InitializePluginActivatorCommand. One SessionManager instance exists for each instance of the core component.

The created plugin activator is added to the context during execution of an operation. The plugin activator are accessed by commands from the context.

PluginActivator.createSession(..) semantics:

Semantics for PluginActivator.createSession(..):

  1. Get resource
  2. Get credential fra credential provider
  3. Get resource-factory from resource
  4. Create session using resource-factory
  5. Connect to session, using resource + credential
Sequence diagram illustrating PluginActivator.createSession(..) semantics
PluginActivator.createVisitor(..) semantics

Semantics for PluginActivator.createVisitor(..):

  1. Get resource
  2. Get credential fra credential provider
  3. Get resource-factory from resource
  4. Get operation from context
  5. Create visitor using resource-factory

Session

Defines the interface for a session to an external resource.

NullSessionImpl

Null implementation of the Session interface. Can be used by plugins which do not require a session based access to its managed resource.

SessionException

Defines exception class for the session package.
