Overview

The WebLogic scripting tool (WLST) plugin implements functionality to execute WebLogic scripts (WLST). For more information about WLST, please refer to the pages:

  • WLST documentation home page for WebLogic 10.3.5 (11g).
  • WLST documentation home page for WebLogic 12.1.1 (12c).

The plugin supports execution of scripts against these releases of WebLogic:

  • 9.2.x
  • 10.3.x (11g)
  • 12.1.x (12c)

Usage

Go to the Plugin Usage page for instructions on using the plugin.

Supported operations

The plugin supports the default operations:

  • test (from the test workflow). When the plugin is invoked with this operation, the content of the module model (e.g. WLST script configuration) is validated with the home directory of the targeted resource (e.g. a product home directory).
  • deploy-configuration (from the deployment workflow). When the plugin is invoked with this operation, the plugin executes one or more WLST scripts.
  • undeploy-configuration (from the deployment workflow). When the plugin is invoked with this operation, the plugin executes executes one or more WLST scripts.

For more information about the default operations, refer to the Operation and workflow reference.

Plugin schema

The plugin defines a schema named WebLogic scripting tool (WLST) plugin schema for definition of models in module model files. The schema defines the namespace http://pineapple.dev.java.net/ns/plugin/weblogic_scriptingtool_1_0. For more information about where the schema can be found, refer to the Schema locations for plugins page.
