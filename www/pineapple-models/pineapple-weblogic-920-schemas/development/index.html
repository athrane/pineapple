Development documentation

Schema location

The project uses multiple schemas for definition of models. The main schema file domain_9_2.xsd defines the namespace:

  • http://www.bea.com/ns/weblogic/920/domain.

Three auxiliary schema files defines the namespaces:

  • security_9_2.xsd defines http://www.bea.com/ns/weblogic/90/security.
  • wls_9_2.xsd defines http://www.bea.com/ns/weblogic/90/security/wls.
  • xacml_9_2.xsd defines http://www.bea.com/ns/weblogic/90/security/xacml.

Finally, the schema file diagnostics_9_2.xsd defines the namespace:

  • http://www.bea.com/ns/weblogic/90/diagnostics.

The schemas are located in the folder: src/main/resources.

Adding the schemas to the main project web site

During the site phase the schemas are copied to the main site in the pineapple-project. The schemas is copied to the directory /ns.

This is implemented in the project pom.xml.

Generation of classes from XML schemas

The project uses XMLBeans through the Maven 2 plugin xmlbeans-maven-plugin to generate classes from XML schemas.

Package and type names are not customized.
