How-to: Use the AbstractDirectoryTestCase class for tests who requires disk access

com.alpha.testutils.AbstractDirectoryTestCase is an abstract test case class which create runtime directory for each method and cleans up afterwards (can be turned off for debugging purposes).

To use AbstractDirectoryTestCase implement this in the test class:

  • The test class must extend AbstractDirectoryTestCase.
  • The start of setup() method should invoke super.setup().
  • The end of teardown() method should invoke super.teardown().
  • Access the inherited class fields to access the test directory or the name of current test method:
    • testDir - The current directory where the test is run.
    • testMethodName - The name of the current test method.
    • deleteDirectories - Set to false to disable deletion of directories after test execution.

Example

public class GoodTest extends AbstractDirectoryTestCase {

  @Before
  public void setUp() throws Exception {
  
        // invoke to create directory 
    super.setUp();
    
    // remaining setup for this test case
    ...
    
  }    
    
  @After
  public void tearDown() throws Exception {
  
    // clean up for this test case
    ...

        // invoke to delete directory and files    
    super.tearDown();  
  }

  @Test
  public void testUsingDirectory() {
  
        // using the current test directory
        file somefile = new File this.testDir, "file-used-in-test.xml" );
        
        // do the test
        ...     
  }      

}
