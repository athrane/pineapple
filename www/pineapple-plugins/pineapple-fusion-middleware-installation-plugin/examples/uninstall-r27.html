How-to: Install JRockit R27

Overview

This example defines the installation instructions for installation of JRockit R27 on a local machine.

Three steps are needed to configure the plugin to work:

  • Define resource which installation on the local machine named "jrockt-install".
  • Define a credential which contains null information since the plugin doesn't require any credentials.
  • Define the installation instruction stanza which directs the installation.

..And finally Pineapple should be invoked to run a test using the module.

For information about how to define resource, credentials and modules, refer to the plugin usage page.

Defining the installation instructions

Starting the installation instructions definition

The basic module module file looks like:

<?xml version="1.0" encoding="UTF-8"?>
<mmd:models xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
  xmlns:mmd="http://pineapple.dev.java.net/ns/module_model_1_0"                 
  xmlns:jip="http://pineapple.dev.java.net/ns/plugin/jrockit_installation_1_0" />
  
  <mmd:model target-resource="jrockit-install">         
    <mmd:content>
    </mmd:content>                                                                                                                                                                              
  </mmd:model>
</mmd:models>

..here is the assumption that the defined resource is named jrockit-install. Then the skeleton of the installation instruction is added:

<?xml version="1.0" encoding="UTF-8"?>
<mmd:models xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
  xmlns:mmd="http://pineapple.dev.java.net/ns/module_model_1_0"                 
  xmlns:jip="http://pineapple.dev.java.net/ns/plugin/jrockit_installation_1_0" />
  
  <mmd:model target-resource="jrockit-install">         
    <mmd:content>
      <jip:jrockit>
      </jip:jrockit>      
    </mmd:content>                                                                                                                                                                              
  </mmd:model>
</mmd:models>

Added the R27 instructions:

The final step is to define r27 installation instruction: 

<?xml version="1.0" encoding="UTF-8"?>
<mmd:models xmlns:xs="http://www.w3.org/2001/XMLSchema-instance"
  xmlns:mmd="http://pineapple.dev.java.net/ns/module_model_1_0"                 
  xmlns:jip="http://pineapple.dev.java.net/ns/plugin/jrockit_installation_1_0" />
  
  <mmd:model target-resource="jrockit-install">         
    <mmd:content>
      <jip:jrockit>
        <jip:r27 installer="./bin/jrmc-3.1.0-1.5.0-windows-ia32.exe" 
          target-directory="c:\jrockit\R27.6.3-1.5.0_17"
          install-public-jre="false" />
      </jip:jrockit>      
    </mmd:content>      
  </mmd:model>
</mmd:models>

with the mandatory attributes:

  • installer which defines where the installer is located.
  • target-directory which defines where the JVM should be installed.
  • install-public-jre which defines whether the JVM should be installed as a public JRE.

Invoke Pineapple to install the JVM

Start your Pineapple client of choice:

  • Select the module where the installation is where just defined.
  • Select the environment for the model where the installation where just defined.
  • Invoke the deploy-configuration operation.
